## 📊 METRICS DEFINITIONS

| Metric | Formula | Example |
|--------|---------|---------|
| **Win rate %** | Wins / Total trades × 100 | 8/12 = 67% |
| **Profit factor** | Total wins / Total losses | $360 / $190 = 1.89 |
| **PnL** | Total wins - Total losses | $360 - $126 = +$234 |
| **PnL %** | PnL / Starting balance × 100 | $234 / $10k = 2.34% |
| **Max drawdown** | Lowest point from peak | -$450 (lowest equity) |
| **Sharpe ratio** | Return / Volatility | (calculated weekly) |
| **Exposure %** | Open position $ / Account | $320 / $10k = 3.2% |

## 🚨 RED FLAG ALERTS

**MASTER gets notified IMMEDIATELY if:**
```
🔴 CRITICAL:
  ├─ Daily loss reaches -1.5% → STOP TRADING
  ├─ SL failure detected → Emergency alert
  ├─ Bot crashes (uptime <99%) → Restart + alert
  ├─ Exposure >5% → Reduce positions
  └─ API connection lost → Reconnect + alert

🟡 WARNING (monitor, report):
  ├─ Win rate drops <55% in 24h
  ├─ Consecutive 3 losses
  ├─ Drawdown >3% intraday
  ├─ Order latency >2 seconds
  └─ Unusual market volatility
```

**Format:**
```
[ALERT - CRITICAL]
2026-03-24 14:32:15 GMT+1
Issue: Exposure exceeded 5% (currently 5.8%)
Action: Closed 1 position (XRPUSDT)
Status: Now 4.2% ✅
MASTER notification: SENT
```

## 💡 BEST PRACTICES

1. **Report early** — don't wait until deadline
2. **Be precise** — exact numbers, not estimates
3. **Flag trends** — "win rate declining 3 days in row" → alert
4. **Celebrate wins** — acknowledge good days
5. **Root cause analysis** — "why did we lose?" not just "we lost"
6. **Actionable insights** — "suggests X" not just "data says Y"
